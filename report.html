Ajustando um modelo VAR ao Dolar Futuro¶

Uir Caiado. 05 de Setembro, 2016

Resumo

Uma das principais necessidades quando falamos trading realizar algum tipo de previso sobre o estado futuro do instrumento operado para que possamos posicionar nossas ofertas. Como um dos objetivos da anlise de sries temporais multivariadas a realizao de forecasts, nesta atividade vou implementar um modelo desta classe chamado Vetor Auto Regressivo (VAR). Vou ajustar o modelo s informaes de book do DOL.

1. Introdução¶

Quando operamos, frequentemente preciasamos tomar decises como manter uma posio ou no, comprar, vender, no fazer nada. Se possurmos uma srie temporal de observaes relacionadas varivel de interesse (o instrumento ou spread operado, por exemplo) e se estes dados possurem informao sobre a dinmica dela, Lutkepohl afirma que razovel utilizarmos para a previso alguma funo dos dados j coletados, de maneira que:

$$\hat{y}_{K,\, T+1} = f \left( y_{1, T}, y_{2, T}, y_{1, T-1}, \dots \right)$$

Onde $y_{k, t}$ uma varivel aleatria, onde $t$ denota o ndice dela no tempo e $k$, o ndice da prpria varivel. A equao de forecast acima implica que pode haver interdependncia entre as varaveis utilizadas. Estas multiplas variveis podem ser os retornos de diferentes ativos, variveis macro-econmica e etc.

Dado que a srie temporal um conjunto de variveis aleatrias, assumimos que essa srie gerada por um processo estocstico com um espao de probabilidade $\left ( \Omega, \, \mathscr{F}, \, P \right )$ associado, onde $\Omega$ o espao amostral, $\mathscr{F}$ ou $\sigma-algebra$ representa todos os subconjuntos possveis de $\Omega$ e $P$, a medida de probabilidade. Definimos que uma variel aleatria uma funo que associa elementos de um espao amostral $\Omega$ a valores no conjunto de nmeros Reais.

Assim, assumindo que os nossos dados so realizaes de variveis aleatrias, que a mesma processo gerador de dados prevalece sobre todo o perodo $T$ (ainda que no conhecido), que funes lineares (nos parmetros) so relativamente fceis de lidar, podemos escrever um processo auto regressivo vetorial de forma que:

$$y_t = \upsilon + A_q y_{t-1} + \dots + A_p y_{t-p} + u_t$$

onde $y_t := \left( y_{1t}, \dots, y_{Kt}\right)'$ um vetor de variveis aleatrias, $\upsilon := \left( \upsilon_{t}, \dots, \upsilon_{K}\right)'$ um vetor de constantes, $A$ uma matriz quadrada de parmetros e $u_t = \left( u_{1t}, \dots, u_{Kt}\right)$ uma sequncia dops erros de previso, tidos como iid com mdia $0$. Segundo notas de aula, isto equivalente a dizer que toda a informao til contida no $\sigma-algebra$ foi incorporada ao $y_t$ e no existem erros sistemticos nos forecasts.

Neste trabalho vou estimar os parmetros do modelo acima, que tambm chamado de modelo de vetor auto regressivo (VAR). Dado que um AR, esperado que os dados utilizados como input sejam estacionrios (primeiro e segundo momento invariante) e que o modelo produzido seja estvel (todos os autovalores de $A$ possuem mdulo menor do que 1).

Depois vou utiliz-lo para realizar previses (aqui tambm referenciados como forecast) e estimar os critrios de seleo: erro de predio (FPE), critrio de informao de Akaike (AIC), critrio de Hannan-Quinn e critrio de Schwarz. Por fim, vou ajustar o modelo alguns dados atuais de mercado.

Onde as colunas representam $K$ e as linhas, $T$. Tambm vamos assumir que os dados utilizandos so estacionrios (o que realmente parece, olhando para o grfico). Definindo:

$$ \begin{aligned} Y &:= \left( y_1, \dots, y_T \right) \;\;\;\;\; &\left(K \times T \right)\\ B &:= \left( \upsilon,\, A_1 \dots, A_p \right) \;\;\;\;\; &\left(K \times (Kp + 1) \right)\\ Z_t &:= \left( 1, y_t, \dots, y_{t-p+1} \right)' \;\;\;\;\; &\left(1 \times (Kp + 1) \right)\\ Z &:= \left( Z_0, \dots, Z_T \right) \;\;\;\;\; &\left((Kp + 1) \times T \right) \end{aligned} $$

Utilizando notao de mtraiz, reescrevemos o var como sendo da forma $Y = B Z + U$, implicando que os residuos so da forma $U = Y - BZ$. Para determinar o estimador de mnimos quadrados (Econometric Methods with Application in Business and Economics, pg. 121), escrevemos a soma do quadrado dos resduos como:

$$ \begin{aligned} S(b) &= \sum U^2 = U'U = \left( Y - BZ \right)' \left( Y - BZ \right)\\ &= Y'Y - Y'BZ - B'Z'Y + B'Z'ZB\\ \end{aligned} $$

Como $Y'ZB = B'Z'Y$, podemos reescrever a equao acima como:

$$S(b) = Y'Y - 2B'Z'Y + B'Z'ZB$$

Derivando em relao a $B$ e igualando a zero, finalmente chegamos que:

$$ \begin{aligned} S(b) &= \sum U^2 = U'U = \left( Y - BZ \right)' \left( Y - BZ \right)\\ &= Y'Y - Y'BZ - B'Z'Y + B'Z'ZB\\ \end{aligned} $$

2.2. Forecast¶

Segundo notas de aula, para se afirmar o que ocorrer no futuro $\left ( y_1, ..., y_k \right )$, tendo um processo $VAR(p)$ ajustado a um conjunto de dados $\Omega_t = \{ y_s \mid s \leq t \}$ para um horizonte de tempo $h$, precisamos determinar qual o forecast timo determinando aquele que minimiza uma funo custo associada seus erros (quadrticos mdios - MSE na sigla em ingls). O preditor que minimiza estes erros a esperana condicional (Lutkepohl, p. 33)

$$\mathbf{E}\left [ y_{t+h} \right ] := \mathbf{E}\left [ y_{t+h} \mid \Omega_t \right ] = \mathbf{E}\left [ y_{t+h} \mid \{ y_s \mid s \leq t \} \right ]$$

Lutkepohl ainda demonstra que a otimizao da esperana condicional impica que:

$$ \mathbf{E}\left [ y_{t+1} \right ] = \upsilon + A_1 y_t + ... + A_p y_{t-p + 1} \\ \mathbf{E}\left [ y_{t+2} \right ] = \upsilon + A_1 \mathbf{E}\left [ y_{t+1} \right ] + A_2 y_t + ... + A_p y_{t-p + 2} \\ \vdots $$

2.3. Intervalo de Confiança¶

Segundo Lutkepohl (p. 38), o erro de previso (e consequentemente, o intervalo de confiana) pode ser obtido atravs da matriz de covarincia dos erros (ou, da sigla em ingls, MSE matrix). Porm, quando se deseja realizar forecasts para mais de um perodo, necessrio definir tambm a matriz de coeficientes de Mdia Mvel (MA), que para um VAR(2) fica:

$$ \phi_1 = A_1 \\ \phi_2 = \phi_1 A_1 + A2 \\ \phi_3 = \phi_2 A_1 + \phi_1 A2 \\ \vdots \\ \phi_i = \phi_{i-1} A_1 + \phi_{i-2} A_2 $$

Assim, a matriz MSE de forecast so obtidas recursivamente aplicando

$$ \Gamma_y(0) = \Sigma_y = \sum_{i=0}^{\infty} \phi_i \Sigma_u \phi_i^{'} $$

Sendo que: $$ \Sigma_y (1) = \Sigma_u \\ \Sigma_y (2) = \Sigma_u + \phi_{1} \Sigma_u \phi_{1}' \\ \Sigma_y (3) = \Sigma_y (2) + \phi_{2} \Sigma_u \phi_{2}' \\ \vdots $$

O $h$ em $\Sigma_y (h)$ se refere a quantos passos para frente se aplica a funo de forecast. Como o modelo de VAR assume que os erros $u_t \sim N \left( 0, \, \Sigma_y(h) \right)$, podemos assumir que o erro de forecast tambm normalmente distribudo. Com este pressuposto, podemos definir um intervalo de confiana na forma: $$\left[ y_{k, \, t} (h) - z_{(\alpha / 2)} \sigma_k (h), \, \, \, \, \, y_{k, \, t} (h) + z_{(\alpha / 2)} \sigma_k (h) \right]$$

Onde $\sigma(h)$ a raiz quadrada do k-simo elemento da diagonal $\Sigma_y(h)$. Assim, seguindo exemplo do livro, se tivermos: $$ \Sigma_y (1) = \begin{pmatrix} \textbf{2.25} & 0 & 0 \\ 0 & \textbf{1.0} & .5 \\ 0 & .5 & \textbf{.74} \end{pmatrix} $$

Os intervalos de confiana so dados por:

$$ y_{1, t} (1) \pm z_{(\alpha / 2)} \sqrt{2.25} \\ y_{2, t} (1) \pm z_{(\alpha / 2)} \sqrt{1.0} \\ y_{3, t} (1) \pm z_{(\alpha / 2)} \sqrt{.74} $$

Assim, vamos calcular os intervalos de confiana para 3 perdos para frente e comparar com a implementao do benchmark.

2.4. Selecionando a Ordem do VAR¶

Como visto, alm dos dados anteriores, o modelo VAR tem mais uma varivel: A ordem do processo auto regressivo. Segundo notas de aula, devemos escolher esta ordem de maneira que as projees tenham a melhor preciso (ou o menor erro quadrtico mdio). Lutkepohl argumenta que no h uma nica maneira de se fazer isso. Uma opo, por exemplo, utilizar o critrio de erro de predio final (FPE), definido como:

$$FPE(m) = \left[ \frac{T + Km + 1}{T - Km -1} \right]^{K} \left | \Sigma_u(m) \right | $$

Onde $\left | . \right |$ o determinante e $m$ a ordem testada (p. 147, Lutkepohl). Outro critrio O Critrio de informao de Akaike (AIC), definido como:

$$AIC(m) = \ln \left | \Sigma_u(m) \right | + \frac{2mK^2}{T}$$

Uma terceira opo o Critrio de Hannan-Quinn, que possui propriedades amostrais interessantes (consistncia, neste caso). Ele da forma:

$$HQ(m) = \ln \left | \Sigma_u(m) \right | + \frac{2\ln \ln T}{T}mK^2$$

A ltima opo implementada ser outro com a mesma propriedade assinttica do critrio anterior. O critrio de Schwarz (SC ou BIC), da forma:

$$SC(m) = BIC(m) = \ln \left | \Sigma_u(m) \right | + \frac{\ln T}{T}mK^2$$

3. Aplicando o modelo VAR¶

Nesta seo vamos tentar aplicar o modelo dado de alta frequncia de Dolar Futuro. Os dados sero agrupados em intervalores de 10 segundos e, ao invs de ajustar o modelo aos retornos, o modelo ser aplicado primeira diferena das variveis selecionadas.

3.1. Order Flow Imbalance¶

Cont at all argumenta que os eventos do book - ordens a mercado, ordens limite e cancelamentos - impactam na dinmica do preo do ativo e que este impacto pode ser modelado atravs de uma nica varivel: o order flow imbalance (OFI), que representa o net de quantidade que entrou e saiu das filas do bid e do ask. Assim, para cada evento $e_n$ no Book de ofertas:

$$e_n = \mathbb{1}_{P_{n}^{B} \geq P_{n-1}^{B}} q^{B}_{n} - \mathbb{1}_{P_{n}^{B} \leq P_{n-1}^{B}} q^{B}_{n-1} - \mathbb{1}_{P_{n}^{A} \leq P_{n-1}^{A}} q^{A}_{n} + \mathbb{1}_{P_{n}^{A} \geq P_{n-1}^{A}} q^{A}_{n-1}$$

Onde $q^{B}_{n}$, $q^{A}_{n}$ se refere a quantidade no melhor preo no Bid e no Ask, respectivamente no perodo atual $n$. O subscrito $n-1$ se refere ao estado anterior. $\mathbb{1}$ uma funo Indicadora. Como os eventos que afetam o book acontecem em tempos $\tau_n$ aleatrios, definimos $N(t) = \max \{ n \mid \tau_n \leq t \}$ o nmero de eventos que ocorreram entre $\left [ 0, \, t\right ]$. A varivel OFI definida sobre o intervalo $\left [ t_{k-1}, \, t_k\right ]$ e a soma de cada $e_n$ que ocorreu neste perodo, de maneira que:

$$OFI_k = \sum^{N(t_k)}_{n=N(t_{k-1})+1} e_n$$

Eles alegam que esta varivel consegue explicar utilizando uma relao linear simples a mudana no mid-price em curtos espaos de tempo. A intuio, segundo eles, que "precisa de volume para movimentar o preo". A mudana no mid-price, no estudo deles, definido em ticks. Assim:

$$\Delta P_k = \frac{\left( P_k - P_{k-1}\right)}{\delta}$$

Onde $\delta$ o tamanho do tick do instrumento analisado.

def measure_e_n(row, last_best):
    '''
    Measure the e_n of the current event
    :param row: dictionary. current row from the file
    :param last_best: tuple. best price and best quantity
    '''
    e_n = 0
    if row['Type'] == 'BID':
        e_n += (row['Price'] >= last_best[0]) * row['Size']
        e_n -= (row['Price'] <= last_best[0]) * last_best[1]
    elif row['Type'] == 'ASK':
        e_n -= (row['Price'] <= last_best[0]) * row['Size']
        e_n += (row['Price'] >= last_best[0]) * last_best[1]
    return e_n

            f_first_price = row['Price']
        # nao preciso lidar com os trades, pois jah esta refletido no bid e ask
        if row['Type'] in ['BID', 'ASK']:
            # converte string para float
            row['Price'] = float(row['Price'].replace(',', '.'))
            row['Size'] = float(row['Size'])
            f_current_time = sum([float(x)*60**(2.-i_aux)  for i_aux, x in enumerate(row['Date'][-8:].split(":"))])
            if f_current_time > f_next_time:
                # imprime resultado
                s_time = convert_float_to_time(f_next_time)
                f_change = 0
                f_logrtn = 0.
                if f_mid:
                    f_curent_mid = (d_best_price['ASK'][0] + d_best_price['BID'][0])/2.
                    f_change = int((f_curent_mid - f_mid)/0.5)
                    f_logrtn = np.log((f_curent_mid/f_mid))
                f_mid = (d_best_price['ASK'][0] + d_best_price['BID'][0])/2.
                s_out = '{}\t{}\t{}\t{}\n'.format(s_time, f_ofi, f_change, f_logrtn)
                fw_out.write(s_out)
                # zera valor
                f_ofi = 0
                 # imprime de 10 em 10 s
                f_next_time  = (int(f_current_time/f_min_time) + 1)*f_min_time
            elif abs(f_current_time - f_next_time) > 3600:
                # new day
                f_next_time = 9 * 3600
                f_mid = None
                f_ofi = 0
            # compara com valor anterior
            last_best = d_best_price[row['Type']]
            f_e_n = measure_e_n(row, last_best)
            # atualiza last best
            d_best_price[row['Type']] = (row['Price'], row['Size'])
            row['Date'] = row['Date'][-8:]
            f_ofi += f_e_n
